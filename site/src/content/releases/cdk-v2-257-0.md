## 概要

このリリースでは、L1 CloudFormation リソース定義の最新化と、ポリシー検証エラー時の合成挙動を制御する新しいコンテキストキー `@aws-cdk/core:failSynthOnValidationErrors` が追加されました。L1 リソース定義の更新に伴い、`AWS::NeptuneGraph::GraphSnapshot` の `GraphIdentifier` プロパティが必須化されるという破壊的変更が含まれています。

**リリース:** [v2.257.0](https://github.com/aws/aws-cdk/releases/tag/v2.257.0)

---

## 新機能

### L1 CloudFormation リソース定義の更新 ([#37955](https://github.com/aws/aws-cdk/pull/37955))

**この機能でできること:**
- `@aws-cdk/aws-service-spec` の最新 CloudFormation Resource Schema に追従し、各サービスの L1 (Cfn) リソースに新しいプロパティ・型・enum 値が追加されました。
- 主な追加・更新の例:
  - **ARC Region Switch:** `AWS::ARCRegionSwitch::Plan` に Lambda Event Source Mapping 構成 (`LambdaEventSourceMappingConfig`) と関連型 (`EventSourceMapping` / `LambdaEventSourceMappingConfiguration` / `LambdaEventSourceMappingUngraceful`) が追加され、`ExecutionBlockType` enum に `LambdaEventSourceMapping` が加わりました。
  - **Neptune Graph:** `AWS::NeptuneGraph::GraphSnapshot` の `GraphIdentifier` プロパティが必須化されました (詳細は後述の破壊的変更参照)。
  - その他、各サービスの最新スキーマに基づく型定義の同期。

**使用例 (ARC Region Switch の Lambda Event Source Mapping ステップ):**

```typescript
import * as arcrs from 'aws-cdk-lib/aws-arcregionswitch';

new arcrs.CfnPlan(this, 'MyPlan', {
  name: 'MyRegionSwitchPlan',
  primaryRegion: 'us-east-1',
  recoveryApproach: 'activeActive',
  regions: ['us-east-1', 'us-west-2'],
  workflows: [{
    workflowTargetAction: 'activate',
    steps: [{
      name: 'DisableLambdaESM',
      // 新しい ExecutionBlockType として LambdaEventSourceMapping が利用可能に
      executionBlockType: 'LambdaEventSourceMapping',
      executionBlockConfiguration: {
        lambdaEventSourceMappingConfig: {
          action: 'disable',           // "enable" | "disable" (必須)
          timeoutMinutes: 60,           // タイムアウト分数 (デフォルト: 60)
          regionEventSourceMappings: {
            'us-east-1': {
              arn: 'arn:aws:lambda:us-east-1:123456789012:event-source-mapping:abc-def', // 必須
              // crossAccountRole: 'arn:aws:iam::...:role/...',  // クロスアカウント時のみ
              // externalId: 'external-id',
            },
          },
          ungraceful: {
            behavior: 'skip',           // "skip" (デフォルト)
          },
        },
      },
    }],
  }],
});
```

**ポイント:**
- L1 リソースは公開 CloudFormation Resource Schema から自動生成されるため、AWS 側のスキーマ更新が CDK の型定義に反映されます。
- 多くのサービスで型の追加・拡張が含まれていますので、利用しているサービスの L1 リソースを使う際は、IDE 補完やリリースノート差分を確認することをおすすめします。

### `@aws-cdk/core:failSynthOnValidationErrors` コンテキストキーの追加 ([#37909](https://github.com/aws/aws-cdk/pull/37909))

**この機能でできること:**
- ポリシー検証 (Policy Validation) でエラーが発生した際に、合成 (synthesis) 側で行っていた **stderr 出力**と **`process.exitCode = 1` 設定**を抑止できる新しいコンテキストキー `@aws-cdk/core:failSynthOnValidationErrors` が追加されました。
- これにより、CDK CLI の `validate` コマンド側で検証レポートの表示・失敗ハンドリングを完全に主導できるようになり、合成と CLI の二重出力・早期プロセス終了といった問題を回避できます。

**使用例:**

```json
// cdk.json
{
  "context": {
    // false に設定すると、合成側ではコンソール出力も exitCode 設定も行わず、
    // JSON レポート (policy-validation-report.json) のみが Cloud Assembly に書き出される。
    // 呼び出し側 (CLI など) が出力と失敗ハンドリングを引き受ける前提の動作。
    "@aws-cdk/core:failSynthOnValidationErrors": false
  }
}
```

```typescript
// または App コンテキストで設定
import { App } from 'aws-cdk-lib';

const app = new App({
  context: {
    '@aws-cdk/core:failSynthOnValidationErrors': false,
  },
});
```

**ポイント:**
- **デフォルト (未設定または `true`):** 従来どおり、JSON レポートを Cloud Assembly に書き出し、人間可読なレポートを stderr に出力し、失敗時に `process.exitCode = 1` を設定します。
- **`false`:** JSON レポートのみを Cloud Assembly に書き出し、コンソール出力も exit code 設定も行いません。CLI など呼び出し側が出力・失敗判定を担う前提の動作です。
- 「Policy Validation Successful!」の成功メッセージは廃止され、no news is good news の方針となりました。
- 失敗時メッセージも `Validation failed. A copy of this report can be found in '<path>'` に変更されています。
- このキーは `cx-api` で公開されており、CDK CLI 側 ([aws/aws-cdk-cli#1515](https://github.com/aws/aws-cdk-cli/issues/1515)) からも参照されます。

## 破壊的変更

### `AWS::NeptuneGraph::GraphSnapshot` の `GraphIdentifier` が必須化 ([#37955](https://github.com/aws/aws-cdk/pull/37955))

**変更前:**

```typescript
import * as neptunegraph from 'aws-cdk-lib/aws-neptunegraph';

// graphIdentifier はオプショナルだった
new neptunegraph.CfnGraphSnapshot(this, 'Snapshot', {
  snapshotName: 'my-snapshot',
});
```

**変更後:**

```typescript
import * as neptunegraph from 'aws-cdk-lib/aws-neptunegraph';

// graphIdentifier が必須プロパティになった
new neptunegraph.CfnGraphSnapshot(this, 'Snapshot', {
  graphIdentifier: 'g-abcd1234',  // 必須: スナップショット元の Graph ID
  snapshotName: 'my-snapshot',
});
```

**移行方法:**
- `CfnGraphSnapshot` を利用している箇所で `graphIdentifier` プロパティを必ず指定してください。
- L1 リソースは公開 CloudFormation Resource Schema から自動生成されており、CloudFormation の実態に合わせた変更です。本来必須であった項目が型定義上も必須になった、というのが今回の変更の本質です。

## まとめ

L1 リソース定義の最新化と、CLI 連携を見据えた検証エラー時の合成挙動制御がこのリリースの中心です。Neptune Graph の `GraphSnapshot` を直接利用している場合は破壊的変更への対応が必要なので、アップグレード前に確認してください。
