## 概要

AWS CDK v2.256.1 は、`aws-cdk-lib` の bundled dependencies に関する不具合を修正したパッチリリースです。`npm ci` 実行時に発生していた lock file の不整合エラーが解消されています。

**リリース:** [v2.256.1](https://github.com/aws/aws-cdk/releases/tag/v2.256.1)

## バグ修正

### invalid lock file の修正 ([#37943](https://github.com/aws/aws-cdk/pull/37943))

- **修正内容:** `aws-cdk-lib` のビルド時に実行されていた `strip-nested-bundled-deps.ts` スクリプトが削除されました。
- **影響を受けていた状況:**
  - 以前のリリース ([#37726](https://github.com/aws/aws-cdk/pull/37726)) で、ネストされた bundled dependencies の問題に対処するため、同名の bundled dependency をネストから取り除くスクリプトが追加されていました。
  - しかし、このスクリプトはバージョン競合がある場合にも nested dependency を削除してしまい、結果として `npm ci` が lock file の不整合を検知してエラーとなる問題が発生していました。
  - 具体的には、以下のような依存関係構造で問題が発生していました。

```
aws-cdk-lib (needs semver ^7.7.4)
   +-- @aws-cdk/cloud-assembly-api  (needs semver ^7.8.0)
   |      +-- semver@7.8.0          <-- 削除されてしまっていた
   +-- semver@7.7.4
```

- **修正後:** `cloud-assembly-api` 側で nested dependencies の使用が解消されたため、防御的なスクリプトは不要となり削除されました。これにより `npm ci` が正常に動作するようになります。
- **関連 issue:** [#37929](https://github.com/aws/aws-cdk/issues/37929)、[#37870](https://github.com/aws/aws-cdk/issues/37870)

## まとめ

このリリースは、`aws-cdk-lib` を `npm ci` でインストールした際に発生していた lock file の不整合問題を解消するパッチリリースです。前バージョンでインストールエラーに遭遇していた場合は、本バージョンへの更新を推奨します。
